Change Entity Class: Abilities Page

This tab page of the Change Entity Class dialog allows you to edit the abilities and options of an entity class.

  • Abilities
  • Options
  • See Change Entity Class for a description of other controls in this dialog.

Abilities

This box shows various abilities, depending on the category of the specified entity class.

  • Use the Blocks Attack (Alt+T) option to toggle the “Blocks Attack” ability of the entity class, indicating whether entities of this class obstruct the line of sight for ranged attacks. This applies only to attackers whose “Ranged Attack Mode” is set to Line.
  • The Blocks Attack option has no effect for upgrade classes under the default rules, as upgrades are never placed on the map.
  • Use the Resource Transfer (Alt+R) option to determine whether entities of this class automatically transfer resources to local units (for placed entities) or to their owning faction (for upgrades).
    • None disables automatic resource transfer.
    • Retain enables the ability and retains exhausted entities.
    • Delete enables the ability and deletes exhausted entities.
  • The Resource Transfer option has no effect for unit classes under the default rules, as units never automatically transfer resources.

The remaining abilities are exclusive to unit and terrain classes, respectively.

Unit Class

  • Use the Can Capture Sites (Alt+C) option to toggle the “Capture” ability of the unit class, indicating whether units of this class capture the map site they occupy. The site must contain at least one terrain of a class that also has the “Capture” ability.
  • Use the Can Defend Only (Alt+D) option to toggle the “Defend Only” ability of the unit class, indicating whether units of this class are limited to passive defense. This is a negative ability that prevents combat units from actively attacking during their faction’s turn.
  • Use the Can Heal Damage (Alt+H) option to toggle the “Healing” ability of the unit class, indicating whether units of this class may replenish their strength after taking damage. Enabling this ability does not actually heal injured units but merely informs computer player algorithms that such units are capable of healing.

Terrain Class

  • Use the Units Can Capture (Alt+C) option to toggle the “Capture” ability of the terrain class, indicating whether map sites that contain at least one terrain of this class can be captured by occupying units. At least one unit must be of a class that also has the “Capture” ability.
  • Use the Owner Can Destroy (Alt+D) option to toggle the “Destroy” ability of the terrain class, indicating whether the owner may destroy terrains of this class at will. This ability has no effect if the “Background Terrain” option is enabled.

Options

This box shows additional options, depending on the category of the specified entity class. Only unit and terrain classes provide additional options.

Unit Class

Use the Ranged Attack Mode (Alt+G) option to determine valid targets when a unit of this class performs ranged attacks.

  • Any specifies that any target is valid, including those hidden by fog of war.
  • View specifies that targets must be visible, though not necessarily to the active unit.
  • Line specifies that targets must share a clear line of sight with the active unit. The line of sight is obstructed by any entities between attacker and target that have the “Blocks Attack” ability.

The option has no effect for units that cannot perform ranged attacks.

The remaining options determine the variables used by the default rules. Selecting “(none)” indicates that the default rules should substitute the constant value one. When selecting a variable identifier, make sure that the Variables Page defines a basic value for that variable.

  • Use the Attack Range Attribute (Alt+A) option to indicate which of the unit’s attributes represents its attack range. Zero indicates non-combat units, and one indicates units capable of close combat (mêlée) only.
  • Use the Movement Attribute (Alt+M) option to indicate which of the unit’s attributes represents its movement range, i.e. the maximum cost it can expend on the combined difficulty values of the traversed terrains.
  • Use the Morale Resource (Alt+O) option to indicate which of the unit’s resources represents its current morale. The unit cannot attack unless this value is positive.
  • Use the Strength Resource (Alt+S) option to indicate which of the unit’s resources represents its current strength or health. Taking damage reduces this value, and the unit is destroyed when it reaches zero.

Remarks

The default rules do not currently use the Strength resource, but the described behavior is recommended for custom rules.

Terrain Class

Use the Background Terrain (Alt+B) option to determine whether the terrain class represents background terrain. The terrain stack of every map site must include at least one background terrain. Any stack entries below the topmost background terrain are deleted when the game map is created.

Background terrains should be associated with opaque images that fill an entire map polygon. Enabling this option suppresses the “Destroy” ability, but has no other gameplay effects once the game map has been created.

The remaining options determine the variables used by the default rules. Selecting “(none)” indicates that the default rules should substitute the constant value zero. When selecting a variable identifier, make sure that the Variables Page defines a basic value for that variable.

  • Use the Difficulty Attribute (Alt+D) option to indicate which of the terrain’s attributes represents its difficulty, i.e. the movement cost that counts against a unit’s movement range for each step that contains the terrain.
  • Use the Elevation Attribute (Alt+E) option to indicate which of the terrain’s attributes represents its elevation compared to a scenario-specific standard, e.g. meters above sea level.

Remarks

1. Since pathfinding requires positive movement costs, the default rules adjust the total difficulty of a site’s terrains to the constant value one if the actual sum is zero.

2. The default rules do not currently use the Elevation attribute, but the described behavior is recommended for custom rules.
