Change Identifier Dialog

This dialog allows you to edit the unique identifier of a given scenario element, as specified in the dialog’s title bar.

  • Use the Identifier (Alt+I) field to enter or change the identifier.
  • Choose OK or press Enter to accept any changes made and dismiss the dialog. If an illegal identifier was entered (see below), a notification message will appear and the dialog will stay open.
  • Choose Cancel or press Escape to discard any changes made and dismiss the dialog. Closing the window using X (Close) or Alt+F4 has the same effect.
  • Choose Help or press F1 to show this help page.

Remarks

1. All identifiers must be valid NCNames according to the W3C XML 1.0 specification. That is, they must contain no spaces, and no punctuation other than periods, hyphens, and underscores. Digits are legal but the identifier must start with a letter or underscore.

2. All identifiers must be unique throughout all scenario sections. Hexkit Editor does not currently verify this restriction but it does check for uniqueness within the identifier’s collection, e.g. the list of unit classes for a unit class ID.

3. Choosing OK or pressing Enter while the Identifier field is empty has the same effect as choosing Cancel or pressing Escape. That is, the identifier remains unchanged and the current operation is aborted.
