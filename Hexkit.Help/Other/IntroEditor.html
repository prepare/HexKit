Hexkit Editor

The companion program Hexkit.Editor.exe can be used to create or alter game scenarios. Visit the following sections for more information:

  • Features (below) briefly describes the abilities of Hexkit Editor.
  • Invocation (below) describes the command line parameters Hexkit Editor will accept.
  • Editor Interface describes how to enter commands and read the display in Hexkit Editor.

Features

Hexkit Editor operates on scenario descriptions. A scenario is a list of definitions of game elements divided into a Master section and five subsections: Images, Variables, Entities, Factions, and Areas.

The Master section file defines the scenario proper. Each of the subsections may be stored either as part of the Master section file (“inlined”), or as separate subsection files referenced by the Master section.

External subsection files may be shared among multiple Master section files for easy scenario creation and maintenance. Subsections may be arbitrarily combined, as long as all game elements referred to by any section are properly defined within the same scenario.

Hexkit Editor checks all user input for correctness, but accepts references to nonexistent scenario elements when reading section files. This allows the user to inspect and correct mismatched identifiers when mixing subsections from different scenarios. Note that all invalid references must be corrected before Hexkit Game will accept a scenario.

The rules by which a scenario is played are a combination of built-in rules and an optional user-defined rule script. Hexkit Editor can attach a script file to a scenario but provides no facilities for editing or debugging such files. You will need an external text editor for this purpose, such as Notepad.

The graphical tilesets required for the visual representation of a scenario may be supplied in any common bitmap file format. Hexkit Editor can attach bitmap files to a scenario but provides no facility for the creation or manipulation of such files. You will need an external paint program for this purpose, such as Windows Paint.

Please refer to the User’s Guide, available as a separate download at the Hexkit home page, for more information.

Invocation

You may supply the name of an XML file as a command line parameter to Hexkit.Editor.exe, and you may drag & drop an XML file on the Hexit Editor program icon or on an instance of Hexkit Editor that is already running. Hexkit Editor will examine the XML file and automatically load it if it appears to be a scenario file. Saved game files (which Hexkit Editor cannot load) or unidentifiable files will generate an error message.

When specifying a file name on the command line, you may omit the file extension and use “wildcard” characters (? and *). If you specify a file name without a directory path, Hexkit Editor will search for the file not only in the current directory, but also in the directory tree below the Scenario folder in your Hexkit installation directory.
