Map View

The largest part of the Hexkit Game window is taken up by the map view. This view also appears on the Areas Page of Hexkit Editor, so most of the following description applies to both programs. Pick a subtopic to view:

  • Map Contents
  • Mouse Commands
  • Target Selection

Map Contents

The map view shows the game map as defined by the current scenario and by the current game situation. This map is a rectangular grid composed of squares or hexagons, depending on the scenario. We refer to the individual squares or hexagons as map locations or “sites”.

  • Sites are identified by coordinate pairs (x, y) where x indicates the site’s column index (starting with zero at the leftmost column) and y indicates the site’s row index (starting with zero at the topmost row).
  • Every site contains at least one background terrain type. Other terrain types, if any, visually appear on top of the background terrain; then any units on top of all terrain; and finally any local effects, such as fog or explosions.
  • If multiple units occupy the same site, only the topmost unit is visible.

The View Menu provides additional visual effects, such as a color overlay to indicate terrain ownership. Moreover, some Command Menu items indicate possible target sites by a bright hue.

Mouse Commands

Please refer to the View Menu for other commands related to the map view.

  • While moving the mouse cursor over the map, the coordinates of the current site will appear in the status bar’s Mouse Position field.
  • Left-click on any map site to select it. The selected site is indicated by a white frame. Click on the border area of the map view to deselect the selected site.
  • If the map view is too small to hold the entire map, horizontal and/or vertical scroll bars will appear that you can use to shift the visible portion of the map.
  • If the application window is maximized, you can auto-scroll the map by moving the mouse cursor all the way towards an edge or corner of the screen.
  • When the map view has the input focus, you can rotate the mouse wheel (if present) to change the map view’s Zoom Level.
  • Click the mouse wheel or the middle mouse button (if present) to restore the Standard Zoom Level.

The following functions apply to Hexkit Game only.

  • The contents of the selected site are listed in the Data View.
  • When the selected site contains multiple units, left-click repeatedly on that site to cycle through the entire unit stack. The topmost unit is also selected in the data view.

Note: Windows sends mouse wheel messages to the control that has the input focus, not to the control under the mouse pointer. So if one of the list boxes on the Data View is active, rotating the mouse wheel will not change the zoom level, but rather scroll that list box. In that case, you can return the input focus to the map view by clicking on it.

Target Selection

This subtopic applies to Hexkit Game only.

When the selected site contains friendly units that can attack or move, you can issue these commands directly on the map view, without going through the Command Menu and the corresponding dialogs.

  • All map sites that are possible attack or move targets are automatically highlighted on the map view.
  • Hover the mouse pointer over a highlighted attack target to show the projected losses for attackers and defenders in the Status Bar.
  • Right-click on a highlighted target to immediately attack or move to the target.
  • By default, right-click targeting applies to the selected unit only.
  • Hold down Shift to instead command all eligible units in the selected site. Target highlights, projected losses, and command execution will all change accordingly.

When a right-click command has been executed and the selected unit has no other valid attack or movement targets, the selection automatically jumps to the active faction’s next unit that does have such targets.

Target selection is only available when Hexkit Game awaits a command by the local human player, i.e. while the Command Menu is enabled.

Note: Right-click targeting is a convenient shortcut that does not expose the full power of Hexkit Game commands. Use the regular Attack and Move commands to select specific units within a stack or to perform multi-pronged attacks.
