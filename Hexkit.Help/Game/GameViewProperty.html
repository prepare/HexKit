Property List

The Data View and several Game Dialogs show the properties, or statistics, of a selected entity in a common list format.

Properties include numerical variables describing attributes and resources, and abilities that are either present or absent.

  • Column Property shows the name of each variable.
  • Column Value shows the current basic and modifier values for each variable.
  • Double-click on an entry to show its Variables dialog.

Properties usually appear in the following sequence:

  1. Abilities, if present (see below).
  2. Strength resource for units, if defined.
  3. Any other resources, in alphabetical order.
  4. Any attributes, in alphabetical order.

Separator bars appear between abilities, resources, and attributes.

Value Column

The Value column of a depletable resource shows a colored bar to indicate its state of depletion. A long green bar appears while the resource is fully replenished. As the resource is depleted, the bar grows shorter and changes color, first to yellow and then to red.

Basic values otherwise appear as simple numbers. The display of modifier values depends on their target:

  • Self affects the entity itself and appears in parentheses, e.g. “(+3)”. A basic value for the same variable precedes the modifier value, e.g. “10 (+1)”.
  • Owner affects the resources of the entity’s current owner, if any.
  • Units affects any units, regardless of their owner, except when overriden by an Owner Units modifier.
  • Owner Units affects units with the same owner as the entity, and overrides a corresponding Units modifier.
  • Units and Owner Units normally affect only units in the entity’s current map site. If the entity is an upgrade, these modifiers affect units in the home site of the upgrade’s owner.
  • Units and Owner Units may also appear in ranged variants that affect units within a certain range around the current or home site. The range precedes the modifier value, e.g. “Units 1/+2”. An infinity symbol (∞) indicates a range that comprises the entire map.

When a unit defines an Owner Units modifier for its current map site, that modifier also affects the unit itself and is cumulative with any Self modifier defined by the same unit.

Unit Abilities

An additional “Abilities” row appears for units, followed by a combination of ability indicators.

  • A indicates that the unit can still attack this turn.
  • M indicates that the unit can still move this turn.
  • B indicates that the owner may build units of this class.
  • C indicates that the unit can capture map sites.

A dash indicates that the unit does not have the corresponding ability.

Terrain Abilities

An additional “Abilities” row may appear for terrains.

  • Can Capture indicates that the map site containing the terrain can be captured by occupying units. At least one unit must be of a class that also has the “Capture” ability.
  • Can Destroy indicates that the owner may destroy the terrain at will. Terrains without this ability are indestructible under the default rules.
  • Can Capt./Dest. indicates that the terrain has both abilities.
