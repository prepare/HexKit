Debug Menu

This menu is disabled if no game is active.

  • Save Map Contents writes the current map contents to an Areas XML file. See Remarks for details.
  • Save Scenario File writes the current scenario description to a debug XML file. See Remarks for details.
  • Show Bitmap Catalogs shows the original bitmap catalog created for the current scenario and the scaled bitmap catalog used for the current map view.
  • Show Command Counters shows the total number of game commands and HCL instructions that were executed since the Hexkit Game application was started. These numbers do not include commands and instructions that were read from a saved game.
  • Show Event History shows the Event History dialog.

Remarks

1. Choosing a Debug menu option other than Show Bitmap Catalogs while a human player command is executing or during an interactive game replay will automatically complete the execution or stop the replay, respectively.

2. Save Map Contents shows a standard Windows “Save” dialog, defaulting to an XML file in the Areas section folder, with a name composed of the current scenario title and the current turn and faction indices.

This file contains all data of the current map, as far as it can be represented as scenario data. It can be read into the Areas Page of Hexkit Editor for analysis, editing, and combination with other scenarios.

3. Save Scenario File creates an XML file named Scenario.Debug.xml in the current user’s Hexkit folder. Any existing file with this name will be overwritten.

This file contains all the data that Hexkit Game extracted from the current scenario XML file, including any data that was originally read from subsection files through include pseudo-tags. It can be read back into Hexkit Game with File: New, or into the Master Page of Hexkit Editor with File: Open or Browse.

Scenario authors might use the debug file to make temporary changes to their scenarios, or to check for discrepancies between the debug file and their own XML file(s).

4. Show Command Counters also displays the number of event instructions and ineffectual instructions.

Event instructions show a map view or message event. They always remain in the HCL program but their execution is skipped during silent replays.

Ineffectual instructions had no effect on the current state of the game world, and were therefore eliminated from the HCL program before the associated command is added to the command history.
