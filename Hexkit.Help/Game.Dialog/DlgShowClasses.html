Entity Classes Dialog

This dialog shows all entity classes (units, terrains, effects, and upgrades) in the current scenario.

  • The Category box allows you to select an entity category.
    • Choose Unit (Alt+U) to show all unit classes.
    • Choose Terrain (Alt+T) to show all terrain classes.
    • Choose Effect (Alt+E) to show all effect classes.
    • Choose Upgrade (Alt+G) to show all upgrade classes.
  • The Entity Class List shows all entity classes in the selected category.
    • Left-click on an entry to select it. The remaining dialog controls reflect the selected entry.
  • The Image box shows an image frame of the selected entity class.
    • Use the scroll bar to browse through all frames of multi-frame images.
  • Property List
  • Below appears a description of the selected entity class, or a notice if the scenario does not define any classes in the selected category.
  • Choose OK or press Enter or Escape to dismiss the dialog. Closing the window using X (Close) or Alt+F4 has the same effect.
  • Choose Help or press F1 to show this help page.

Remarks

The Entity Class List may show multiple entries with identical names. This is not an error but an indication that the scenario designer considered different classes similar enough that they should share the same name. Such classes typically differ only in their images but have otherwise identical properties.

Property List

This list shows the properties of the selected entity class, i.e. numerical variables and ability flags.

  • Column Category shows “Attribute”, “Resource”, or “Build Cost”, indicating the category of each variable.
  • Column Property shows the name of each variable.
  • Column Value shows the initial basic or modifier values for each variable.
  • Left-click on an entry to display its Variables dialog.

Modifier values are displayed as in the standard Property List, except that Owner Units modifiers are always shown, even when identical to the corresponding Units modifiers.

Unit Classes

One additional “Ability” row appears for unit classes.

  • Capture indicates that units of the selected class capture any map site they occupy. The site must contain at least one terrain of a class that also has the “Capture” ability. Units without this ability cannot claim sites, although they still benefit from any bonuses provided by the local terrain.

Terrain Classes

Two additional “Ability” rows appear for terrain classes.

  • Can Capture indicates that map sites containing at least one terrain of the selected class can be captured by occupying units. At least one unit must be of a class that also has the “Capture” ability.
  • Can Destroy indicates that the owner may destroy terrains of the selected class at will. Terrains without this ability are indestructible under the default rules.

Remarks

1. The current variable values of an individual entity may differ from the corresponding initial values of its entity class.

2. Build costs are expended by factions to build an instance of the selected entity class. Build costs never have modifier values, and they do not appear in an entity’s Property List.

3. Hexkit Editor defines a third variable category, “Counter”. Variables of this category are intended only for internal use by the rule script, and therefore do not appear in the Hexkit Game GUI.
