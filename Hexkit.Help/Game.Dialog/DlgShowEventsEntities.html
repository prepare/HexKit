Event History: Entities Page

This tab page of the Event History dialog shows the history of all entities in the current game, including those that were already eliminated.

  • The Entity List shows the internal identifiers of all entities.
    • All identifiers are composed of an entity class identifier (up to the final dash) and an instance count (after the final dash).
    • The list entries are sorted alphabetically by entity class identifiers, and numerically by instance counts within the same entity class.
    • Left-click on an entry to select it. The Event List reflects the selected entry.
  • The Event List shows the internal “events” that Hexkit recorded for the selected entity.
    • Turn shows the game turn during which the event was recorded.
    • Event shows the type of the event.
      • Create means that the entity was created.
      • Delete means that the entity was deleted from the game.
      • SetName means that the entity’s instance name has changed.
    • Value shows other data associated with the event. This is usually the active faction at the time of the event, but it is the new instance name for the SetName event.
  • Drag the vertical splitter between Entity List and Event List to adjust their respective share of the current dialog width.
  • See Event History for a description of other controls in this dialog.

Remarks

All entities that were created by the scenario, and possibly by the Faction.Initialize method if it was overriden in the rule script, will appear with a Create event during the first turn and with the first active faction.
